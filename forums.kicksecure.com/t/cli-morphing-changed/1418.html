arraybolt3 December 26, 2025, 6:05pm 8

There‚Äôs a slight misunderstanding here I think.

  • Kicksecure can be used with clearnet. Anonymity features are not intended to land in Kicksecure for the most part, those are for Whonix. Any time you use your web browser or most network-connected applications, they will go over clearnet.
  • There is a specific subset of Kicksecure features that require Tor for non-anonymity purposes. One of those is the time synchronization daemon (sdwdate, which acts as an alternative to ntpd). Another one is software updates (though this is only applicable to non-Qubes platforms).
    • sdwdate works on Qubes OS. It requires Tor because Tor provides an encrypted, authenticated way to get time values from web servers and use those to synchronize the VM‚Äôs time. Traditional methods of time synchronization (ntp) are unencrypted and vulnerable to MITM attacks, and an MITM in time synchronization can allow circumventing SSL (i.e. breaking HTTPS encryption) in some situations.
    • On non-Qubes platforms, software updates in apt are also downloaded over Tor. This is arguably an anonymity feature, but it‚Äôs for a security reason; downloading software updates over clearnet allows a malicious update server to target you (or a group you‚Äôre in) for malware while delivering uninfected software updates to others. By downloading updates over Tor, a malicious update server would have to attack everyone on the Tor network to deliver malware to you, there isn‚Äôt a good way to single you out for the bad updates.
      • Under Qubes OS this usually isn‚Äôt done by default because software updates are downloaded via an update proxy VM. If you set your Kicksecure template‚Äôs update proxy to sys-whonix, you‚Äôll get torified software updates, otherwise software updates will be downloaded via clearnet. If you use a Kicksecure StandaloneVM, its software updates will be torified because it will connect to the Internet directly to get its updates rather than using an updates proxy.
  • Because of these specific Tor-powered features, Tor is a semi-mandatory part of Kicksecure. (edit: changed ‚Äúmandatory‚Äù to ‚Äúsemi-mandatory‚Äù since it is important and usually shouldn‚Äôt be removed, but some users have reasons to remove it anyway.)
  • This does not mean your network traffic typically goes over Tor.

arraybolt3 December 26, 2025, 6:27pm 11

If you really want to get rid of Tor, you should not apt purge it. Use dummy-dependency to remove it instead:

sudo dummy-dependency --purge --yes tor

That will remove Tor without removing kicksecure-qubes-server. Once that command is run, you should run sudo apt install kicksecure-qubes-server to ensure that software updates continue working in the future.

brilliantjeana:

So, then when -server package is being deleted I still keep all of the rest security/privacy benefits that kicksecure provides - correct?

Unfortunately no, since sometimes new security enhancements require the installation of new packages, and without kicksecure-qubes-server, those might not get installed automatically. The workaround above will let you keep the metapackage without keeping Tor.
