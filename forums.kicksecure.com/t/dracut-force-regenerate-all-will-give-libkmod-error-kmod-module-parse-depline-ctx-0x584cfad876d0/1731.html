Just found out that there is stupid mismatch between kernel and/or headers and/or modules, some are .74 (old default kernel) and some are .88 (latest kernel), so how to fix that:

  • Check the current kernel installed:

uname -r

  • Check and Remove old images which are for previous version/s

** Check in modules directory:

ls -1 /lib/modules

Remove old versions:

sudo rm -rf /lib/modules/x.xx.xx+debxx-amd64

** Check in headers directory:

sudo ls -1 /usr/src

Remove Old versions:

sudo rm -rf /usr/src/linux-headers-x.xx.xx+debxx-amd64

** Check in boot directory

sudo ls -lh /boot

Remove old versions:

sudo rm -f /boot/vmlinuz-x.xx.xx+debxx-amd64

and

sudo rm -f /boot/initrd.img-x.xx.xx+debxx-amd64

and

sudo rm -f /boot/initrd.img-x.xx.xx+debxx-amd64

  • Regenerate dracut and Grub

** Dracut

sudo dracut --force --regenerate-all

** Grub

sudo update-grub

Error messages should be cleaned, preferable to reboot after that:

sudo reboot

Done.

Not… quite. Deleting package-owned files can cause weird breakage in some instances (for instance if a package assumes a particular package-owned file exists because the package that owns it is installed).
