So there are flags need to be added to make it compatible with discrete mode:

sudo nano /etc/modprobe.d/nvidia-drm.conf

add:

options nvidia-drm modeset=1 fbdev=1

Then regenerate:

sudo dracut --force --regenerate-all

The next reboot will lead to kernel panic, only way to bootup again it remove the flags from grub:

Press e on grub window, search for the line starting with “linux”, and add to the end of it:

rd.driver.blacklist=nvidia,nvidia_drm,nvidia_modeset,nvidia_uvm modprobe.blacklist=nvidia,nvidia_drm,nvidia_modeset,nvidia_uvm module_blacklist=nvidia,nvidia_drm,nvidia_modeset,nvidia_uvm systemd.unit=multi-user.target

Then F10.

After that it will boot up, and you need to remove the added flags and regenerate dracut.

Note: I have tested that with Debian 13 DVD image with LXQT (default going to be initramfs and xorg) + Nvidia setup as before = it will bootup normally without issues.
